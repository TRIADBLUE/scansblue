Agent Execution Prompt — UI Accent Removal & Button Color Enforcement

Role: Deterministic UI Execution Agent
Authority Level: Absolute (do not reinterpret, redesign, or negotiate scope)

⸻

STOP CONDITIONS
	•	Do not proceed if you are unsure where the styles originate.
	•	If ambiguity exists, locate the source — do not guess, do not defer.

⸻

OBJECTIVE (NON-NEGOTIABLE)
	1.	REMOVE ALL WHITE ACCENTS from ALL RED BUTTONS
	2.	SET ALL “Submit”, “Send”, and form-submission buttons to TRIAD BLUE
	3.	No exceptions. No visual remnants. No partial overrides.

⸻

PART 1 — REMOVE WHITE ACCENTS FROM RED BUTTONS

Definition of “White Accents”

This includes any of the following, anywhere in the UI:
	•	White borders
	•	White outlines
	•	White box-shadows
	•	White inner highlights
	•	White focus rings
	•	White hover overlays
	•	White active/pressed states

Required Action
	•	Identify every CSS rule, token, variable, utility class, or inline style applying white (#FFFFFF, #FFF, rgba(255,255,255,*), or equivalent) to red buttons.
	•	Remove or override them completely.

Explicit Rule

If a button is red, it may NOT contain white in any state.

States that must be checked and fixed:
	•	Default
	•	Hover
	•	Focus
	•	Active
	•	Disabled
	•	Loading

⸻

PART 2 — ENFORCE TRIAD BLUE FOR SUBMIT/SEND BUTTONS

Scope

Applies to ALL buttons whose function is:
	•	Submit
	•	Send
	•	Form submission
	•	Primary action on a form

Includes:
	•	<button type="submit">
	•	Buttons labeled “Submit”, “Send”, “Save & Send”, etc.
	•	Buttons triggered by form handlers or submit events

Color Enforcement
	•	Base color: Triad Blue
	•	Use the platform’s triad blue token if it exists
	•	Otherwise define and use a single explicit hex value consistently
	•	No red variants
	•	No white accents
	•	No gradient unless triad blue-only

States

All states must remain within the triad blue family:
	•	Hover → darker/lighter triad blue
	•	Active → pressed triad blue
	•	Focus → triad blue outline or shadow
	•	Disabled → muted triad blue (not gray, not white)

⸻

PART 3 — IMPLEMENTATION RULES (CRITICAL)
	•	Do not rely on “it should inherit”
	•	Do not leave conflicting rules in place
	•	Do not override visually while keeping broken styles underneath
	•	Fix the source of truth:
	•	Design tokens
	•	Shared button components
	•	Global styles
	•	Utility classes
	•	Theme providers

If styles are duplicated:
	•	Normalize them
	•	Remove redundancy
	•	Ensure one authoritative definition

⸻

PART 4 — VERIFICATION (MANDATORY)

You must explicitly verify:
	•	Red buttons contain zero white pixels
	•	Submit/Send buttons are 100% triad blue
	•	Mobile and desktop match
	•	Light mode and dark mode match

⸻

FAILURE CONDITIONS

The task is NOT complete if:
	•	Any red button still shows white
	•	Any submit/send button is not triad blue
	•	The change only works in some states
	•	The agent claims success without verification

⸻

FINAL OUTPUT REQUIRED

Respond only with:
	1.	What was changed
	2.	Where it was changed
	3.	Why white accents can no longer reappear
	4.	Confirmation that submit/send buttons are triad blue everywhere

No commentary. No explanations beyond facts.
